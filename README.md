# Connect-4
Wanna play connect 4 with your friend? Take turns dropping colored discs into the columns and be the first to line up four in a row to win; either horizontally, diagonally, or vertically. Block your opponent, plan your moves, and claim victory! 

What this program does:
This is a 1 player game against the computer where the goal of the game is to be the first to have four in a row, either, horizontally, vertically, or diagonally. This game includes a welcome screen, an option where the user decides whether they want to be red or yellow, a playable 7 by 6 connect four board, an automatic win detection. By having a good strategy throughout the game, the player can block or play a piece with each turn. 

Why this was made: 

I created this project to challenge myself and improve my understanding of how interactive web applications work. I wanted to build something fun, visual, and hands on that was similar to the games that you would find in an arcade. This program was a mix of UI design, game logic, event handling, DOM manipulation, and problem-solving. Building this program pushed me think both like a programmer and a designer. As I worked through the project, I realized how many different skills go into even a simple game. I had to design a clean user interface, structure the HTML so the screens could switch smoothly, and style everything in a way that felt modern and consistent. On the programming side, I had to think carefully about how to represent the board, how to track turns, how to detect wins in multiple directions, and how to make the computer take its own moves. It wasn’t just about writing code and thinking about the planning, testing, and adjusting process as I coded until the program worked as I had intended.

What I struggled with: 

One of the biggest challenges I faced while building this project was getting all the different parts of the game to work together the way I imagined. In the beginning, nothing on the page responded when I clicked it, and I didn’t understand why. I eventually discovered that even a small mistake, like linking the JavaScript file incorrectly or using the wrong variable name, could cause the entire program to stop running. This forced me to learn how to use the browser console, how to read error messages, and how to trace problems back to the exact line of code that caused them. Being able to detect who had won, especially when the win was horizontal. Imagining what the grid would look like helped me solve the error of the out of bounds as well as making sure that the loops were working correctly with the right positions. 

What I learned:

Even though these struggles were frustrating at times, they ended up being the most valuable part of the project. They taught me patience, problem‑solving, and the importance of reading errors instead of guessing. They also helped me understand that struggling is part of learning and that every bug I fixed made me a stronger programmer. This project also helped me build confidence. At the beginning, the idea of creating a full game from scratch felt overwhelming, but by the end, I realized that I’m capable of building something complex if I take it one step at a time. I learned how to think logically, how to structure my code more clearly, and how to use tools like the browser console to guide me instead of feeling lost. Most importantly, this experience showed me that persistence matters more than getting everything right on the first try. Every obstacle taught me something new, and by pushing through the difficult parts, I ended up with a project I’m proud of and a much stronger understanding of how real web applications work.
